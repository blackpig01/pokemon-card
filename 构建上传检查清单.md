# 微信小程序本地构建与上传检查清单

> 本清单适用于 `pokemon-tracker` 项目，基于 Taro 4.x 框架

---

## 📋 事前准备清单

### 1. 环境检查

| 检查项 | 要求 | 状态 |
|--------|------|------|
| Node.js 版本 | >= 18.0.0 | ☐ |
| npm 版本 | >= 8.0.0 | ☐ |
| 微信开发者工具 | 最新稳定版 | ☐ |
| 项目依赖 | 已完整安装 | ☐ |

**验证命令：**
```bash
# 检查 Node.js 版本
node -v
# 要求输出: v18.x.x 或更高

# 检查 npm 版本
npm -v
# 要求输出: 8.x.x 或更高

# 检查微信开发者工具（图形界面）
# 确认已安装并可正常启动
```

### 2. 项目配置检查

| 配置文件 | 位置 | 检查要点 |
|----------|------|----------|
| `project.config.json` | 项目根目录 | appid 已正确配置 |
| `app.config.ts` | src/ | pages 数组包含所有页面 |
| `app.tsx` | src/ | 入口组件正常 |
| `tsconfig.json` | 项目根目录 | TypeScript 配置正确 |

### 3. appid 配置

打开 `project.config.json`，确认或修改以下内容：

```json
{
  "appid": "wx1234567890abcdef",  // ⚠️ 替换为您的真实AppID
  "projectname": "宝可梦卡牌追踪",
  "description": "宝可梦卡牌价格追踪小程序"
}
```

**如何获取 AppID：**
1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入「开发」→「开发管理」→「开发设置」
3. 找到「AppID（小程序ID）」

## 🔧 依赖安装

### 检查 node_modules 是否存在

```bash
# 进入项目目录
cd pokemon-tracker

# 检查依赖是否已安装
ls node_modules/.bin/taro
# 如果存在，说明依赖已安装

# 如果不存在，执行安装
npm install
```

### 依赖安装命令

```bash
# 使用 npm 安装（推荐）
npm install

# 或使用 yarn（如果已安装 yarn）
yarn install

# 或使用 pnpm（如果已安装 pnpm）
pnpm install
```

**⚠️ 安装时间预估：** 首次安装约 3-5 分钟，视网络状况而定

## 📱 微信开发者工具操作

### 1. 打开微信开发者工具

1. 启动「微信开发者工具」
2. 选择「小程序」项目类型
3. 点击「导入项目」

### 2. 导入项目配置

| 配置项 | 值 |
|--------|-----|
| 项目目录 | `pokemon-tracker/dist` |
| AppID | 您的真实 AppID |
| 项目名称 | `宝可梦卡牌追踪`（可自定义） |
| 编译模式 | 默认模板 |

### 3. 开发者工具设置

点击「详情」→「本地设置」，确认以下选项：

| 设置项 | 建议值 |
|--------|--------|
| ES6 转 ES5 | 开启 |
| 增强编译 | 开启 |
| 自动压缩 | 关闭（方便调试） |
| 代码放大 | 关闭 |

## 🚀 上传流程

### 上传前检查

| 检查项 | 说明 |
|--------|------|
| appid 正确 | 已替换为真实 AppID |
| 代码无语法错误 | 控制台无红色错误 |
| 功能验证通过 | 所有核心功能正常 |

### 正式上传

1. **在微信开发者工具中**
   - 点击右上角「上传」按钮
   - 等待编译完成（约1-2分钟）

2. **填写上传信息**
   ```
   版本号: 1.0.0（示例，可自定义）
   备注:  首次上传-卡盒及周边销售板块+弹幕评论功能
   ```

3. **确认上传**
   - 点击「确认上传」
   - 等待上传完成

4. **在微信公众平台确认**
   - 登录 [微信公众平台](https://mp.weixin.qq.com/)
   - 进入「版本管理」
   - 查看已上传的版本

## ❓ 常见问题与解决

### 问题1：npm 命令找不到

**原因：** Node.js 未正确安装或 PATH 未配置

**解决：**
```bash
# 重新安装 Node.js（推荐使用 nvm 或 fnm）
# 或使用完整路径
"C:\Program Files\nodejs\npm.cmd" install
```

### 问题2：构建失败，提示缺少模块

**原因：** node_modules 未完整安装

**解决：**
```bash
# 删除 node_modules
rm -rf node_modules

# 重新安装
npm install
```

### 问题3：微信开发者工具导入失败

**原因：** appid 格式错误或目录选择错误

**解决：**
1. 确认选择的是 `dist` 目录
2. 确认 appid 格式正确（wx开头的17位字符串）

### 问题4：页面空白或样式错乱

**原因：** 编译类型不匹配

**解决：**
1. 确认是「小程序」模式
2. 点击「工具」→「构建 npm」
3. 重新编译

### 问题5：上传按钮灰色不可点击

**原因：** 项目存在编译错误

**解决：**
1. 查看底部「编译提示」面板
2. 修复红色错误
3. 重新编译

### 问题6：打开项目后一片空白

**可能原因：**
- dist 目录为空（未执行构建）
- app.json 配置错误

**解决：**
```bash
# 1. 确认已执行构建
npm run build:weapp

# 2. 确认 dist 目录有内容
ls dist/

# 3. 确认 app.json 存在
cat dist/app.json
```
